HTML 문서 작성 기본 규칙

  • DTD : HTML5
  • 인코딩 : utf-8

작업 기본 규칙

  • label이 없는 input, select는 title 속성을 넣어주세요.
  • 담당 사이트의 검수는 담당기획자에게 작업된 리스트 페이지를 메일로 요청합니다. 요청할때 각 기획/디자인 담당자, 각 PL을 참조로 보냅니다.
  • Aria 기법을 이용하여 접근성 준수 예) role="text", role="button" 등
  • 전화번호는 필히 <a href="tel:전화번호"></a> 링크 걸어줌.

CSS 코드 작성 규칙

  • 모든 속성은 영문 소문자로만 작성한다.
  • @charset 선언시에만 큰따옴표("")를 사용하고 그외의 속성 파라미터값에는 작은 따옴표('')를 사용한다.
  • background의 url 속성의 파라미터값에는 따옴표를 사용하지 않는다.
  • 마지막 속성의 세미콜론(;)은 삭제한다.
  • css 코드는 작성할때는 들여쓰기를 하지 않는다.
  • 속성간에는 공백, 줄내림을 하지 않으며 모든 속성을 붙여쓰기 한다. 단, scrollbars, CSS3에서 배경그라데이션 색상과 같이 하나의 속성을 여러개의 핵을 써야할 경우에는 허용한다.
  • 중괄호 좌우 공백은 제거한다.
  • 의미있는 객체를 구분하기 위한 빈 줄은 한줄을 초과하지 않게 한다.
  • CSS 최적화를 위해 약식 속성을 이용하여 css 속성이 반복되지 않게 한다.(font, margin, padding, background, border)
  • 약식 속성을 이용하되 속성값이 하나만 있는 경우는 약식 속성을 이용하지 않는다.
  • 최적화를 위해 '0'뒤의 단위는 생략한다.
  • 배경이미지 위치값은 문자식 표현(left, top, center)은 숫자식(0,100% 50%)으로 선언한다.

속성 선언 순서

  • overflow
  • display
  • float
  • position, top, left, right, bottom
  • width, height
  • margin, padding
  • color, font, letter-spacing, white-space
  • text-align, vertical-align, text-decoration, text-indent, border
  • background, z-index
  • 기타(위에 언급하지 않은 나머지 속성들)

단위

z-index

  • z-index 값은 최소 1부터 시작하며 최고 10000이며, 10 단위로 증감한다. 단, 중간에 변수가 생기면 예외로 1단위 값을 지정할 수 있다.
